# 충돌 해결

충돌은 같은 파일을 두 기기에서 동기화 전에 편집했을 때 발생합니다. 예를 들어, 노트북에서 노트를 편집한 후, 노트북이 동기화되기 전에 휴대폰에서 같은 노트를 편집하는 경우입니다.

## 전략 선택

**Settings > Dropbox Sync > Conflict strategy**에서 충돌 처리 방식을 선택합니다.

### Keep both (기본값)

두 버전 모두 저장됩니다. 로컬 버전은 그대로 유지되고, 다른 기기의 버전은 `.conflict` 접미사가 붙어 옆에 저장됩니다:

```
notes/idea.md                          <- 내 버전 (그대로 유지)
notes/idea.conflict-20260306T143200.md <- 다른 기기의 버전
```

여유 있을 때 두 버전을 비교하고, 필요한 내용을 병합한 후, `.conflict` 파일을 삭제하면 됩니다.

### Keep newest

가장 최근에 편집된 버전이 선택됩니다. 이전 버전은 덮어쓰여집니다.

가장 간단한 방식이지만, 두 기기에서 모두 중요한 편집을 했다면 변경사항이 유실될 수 있습니다. 보통 한 기기에서만 편집하는 경우에 적합합니다.

### Ask me

비교 창이 열려서 섹션별로 결정할 수 있습니다.

**텍스트 파일**의 경우 나란히 비교 화면이 표시됩니다:

- 두 기기에서 동일한 섹션은 회색으로 표시됩니다
- 다른 섹션은 강조 표시됩니다 — 클릭하여 **내 버전**, **다른 기기 버전**, 또는 **둘 다**를 선택합니다
- 하단의 카운터에 아직 결정이 필요한 섹션 수가 표시됩니다
- 준비가 되면 **Save**를 클릭합니다

**Keep all local** 또는 **Keep all remote**를 클릭하면 한 번에 모든 충돌을 해결할 수도 있습니다.

**이미지**의 경우 두 버전이 파일 크기와 함께 나란히 표시되어 시각적으로 비교할 수 있습니다.

**기타 파일** (PDF 등)의 경우 파일 크기와 수정 날짜가 표시되어 판단에 도움을 줍니다.

**아직 결정하기 어렵다면?** 시계 아이콘을 클릭하여 이 충돌을 나중으로 미룰 수 있습니다. 다음 동기화 때 다시 나타납니다.

## 팁

- 기기를 전환하기 전에 동기화가 완료될 때까지 몇 초만 기다리면 **충돌은 거의 발생하지 않습니다**
- 같은 파일에서 충돌이 계속 발생한다면, 보통 두 기기에서 동시에 편집하고 있기 때문입니다 — 한 기기에서만 편집해 보세요
- `.conflict` 파일은 일반 파일과 같습니다 — 열고, 편집하고, 삭제할 수 있습니다
