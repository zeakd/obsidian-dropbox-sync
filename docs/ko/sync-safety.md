# 동기화 안전 장치

한 기기에서 파일을 삭제하면, 동기화 플러그인은 다른 기기에서도 해당 파일을 삭제해야 합니다. 이 과정은 위험할 수 있습니다 — 문제가 발생하면 모든 기기에서 파일이 사라질 수 있기 때문입니다. 그래서 이 플러그인은 세 가지 독립적인 안전 장치를 사용합니다.

## 1단계: 의도적인 삭제만 동기화

플러그인은 **Obsidian에서 직접 삭제한 파일**만 Dropbox에서 제거합니다. 삭제하거나 이름을 변경한 모든 파일의 기록을 유지합니다.

실제로 이것이 의미하는 바:

- 오류나 부분 동기화로 인해 파일이 누락된 경우, Dropbox에서 **삭제되지 않습니다** — 대신 다시 다운로드됩니다
- 제외 패턴으로 동기화에서 제외한 파일은 Dropbox에서 **삭제되지 않습니다**
- Obsidian에서 의도적으로 삭제(또는 이름 변경)한 파일만 Dropbox에서 제거됩니다

## 2단계: 대량 삭제 전 확인

한 번에 5개 이상의 파일을 삭제하려는 동기화가 발생하면, 삭제될 파일 목록을 보여주는 확인 창이 나타납니다.

이 장치가 잡아내는 상황:

- 폴더를 실수로 삭제한 경우
- 동기화 상태에 문제가 생긴 경우
- Vault ID를 변경한 경우 (모든 파일이 삭제된 것처럼 보일 수 있음)

삭제가 의도적이라면 **Delete**를 클릭하여 진행하거나, **Skip deletions**를 클릭하여 나머지만 동기화하고 해당 파일은 그대로 둘 수 있습니다.

임계값 (기본값: 5개)은 **Settings > Delete threshold**에서 변경할 수 있습니다.

## 3단계: Dropbox가 삭제된 파일을 보관

Dropbox에서 파일이 삭제되어도 영구적으로 사라지지 않습니다. Dropbox가 휴지통에 보관합니다:

- **Free, Plus 플랜**: 30일
- **Professional, Business 플랜**: 180일

삭제된 파일을 복구하려면 [dropbox.com](https://www.dropbox.com)에 접속하여 사이드바에서 **Deleted files**를 클릭하고, 파일을 찾아 **Restore**를 클릭합니다.

## "지연된(deferred)" 파일이란?

플러그인이 동기화 중에 파일 처리를 건너뛰는 경우가 있습니다:

- 현재 편집 중인 파일은 편집 도중에 덮어쓰지 않습니다
- "나중에" 처리하기로 한 충돌은 다음 동기화로 미뤄집니다

이 파일들은 유실되지 않습니다 — 다음 동기화에서 처리됩니다. 동기화 결과에 지연된 파일이 있는지 알려줍니다.
